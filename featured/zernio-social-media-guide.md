# 🚀 Zernio 社群發布工具 — 快速上手指南

> **適用對象：** 新用戶、剛開通 Zernio 的使用者
> **最後更新：** 2026-07-03

---

## ❓ 什麼是 Zernio？

Zernio 是一個**跨平台社群媒體管理工具**，可以讓你用一個 API 同時發布貼文到：
- Facebook 粉絲專頁
- Instagram
- TikTok

不用逐一手動登入各平台，一次搞定。

---

## ❓ 如何開始使用？

### Step 1：註冊 Zernio 帳號
1. 前往 [Zernio 官網](https://zernio.com) 註冊
2. 完成 Email 驗證

### Step 2：取得 API Key
1. 登入後進入 **Dashboard → Settings → API Keys**
2. 點擊 **Generate New Key**
3. 複製並妥善保存你的 API Key（只會顯示一次）

### Step 3：綁定社群帳號
1. 進入 **Dashboard → Accounts**
2. 依序連接你的：
   - Facebook 粉絲專頁
   - Instagram 帳號
   - TikTok 帳號
3. 授權完成後，系統會自動抓取帳號 ID

### Step 4：驗證連線
使用 API 前，建議先執行驗證流程：

```
1. auth:set → 設定你的 API Key
2. auth:check → 確認連線狀態
3. accounts:list → 查看已綁定的帳號清單
```

確認無誤後即可開始發文！

---

## ❓ 如何發布貼文？

### 基本發文流程

```json
POST /posts
{
  "content": "你的貼文內容",
  "accounts": ["你的帳號ID"],
  "mediaUrl": "圖片或影片網址（選填）"
}
```

### 支援的發文模式

| 模式 | 說明 |
|:-----|:-----|
| **立即發布** | 設定 `publish: true`，貼文會立刻上線 |
| **存為草稿** | 設定 `publish: false`，稍後手動發布 |
| **排程發布** | 使用定時任務（見下方說明） |

---

## ❓ 發文有什麼要注意的？

### ✅ 發文鐵則（必讀！）

1. **段落之間一定要加空行**
   - ❌ 錯誤：`第一段\n第二段\n第三段`
   - ✅ 正確：`第一段\n\n第二段\n\n第三段`
   - 不加空行的話，IG 和 FB 貼文會全部擠在一起！

2. **發文前強制流程**
   ```
   auth:set → auth:check → accounts:list → 發文
   ```
   每次發文前都要確認連線狀態，避免發送失敗。

3. **不同帳號用不同 Key**
   - 如果你管理多個粉專，每個粉專可能對應不同的 API Key
   - 切勿用 A 粉專的 Key 發 B 粉專的內容

4. **來源標註格式**
   - 如果內容來自其他平台，請標註來源：
   - 格式：`@影片來源：平台名稱 頻道名稱`
   - 不需要貼來源連結

## ❓ 排程發布怎麼做？

### 方法：定時任務（推薦）

由於 Zernio 的排程功能（scheduledAt）會存成草稿而非直接排程發布，建議使用**定時任務**來執行：

1. 建立任務腳本（例如 `jobs.json`）
2. 設定執行時間
3. 系統會在指定時間自動執行發文

```
任務類型：一次性定時任務
執行內容：呼叫 Zernio API 發布貼文
```

## ❓ 各平台有什麼限制？

### Facebook
- ✅ 支援文字、圖片、影片
- ✅ 支援留言功能
- ✅ 可發布到粉絲專頁

### Instagram
- ✅ 支援圖片、影片（Reels）
- ⚠️ 必須有圖片或影片才能發布
- ⚠️ 文字貼文無法單獨發布

### TikTok
- ✅ 支援影片發布
- ❌ **不支援留言功能**
- ⚠️ 分潤連結需直接包含在**影片描述**或**影片本體**中

## ❓ 常見問題

### Q：發文失敗怎麼辦？
**A：** 按以下順序檢查：
1. API Key 是否正確？→ 重新執行 `auth:set`
2. 帳號是否綁定？→ 執行 `accounts:list` 確認
3. 內容是否符合平台規範？→ 檢查圖片尺寸、影片格式
4. 網路連線是否正常？→ 稍後重試

### Q：可以一次發布到多個平台嗎？
**A：** 可以！在 `accounts` 陣列中填入多個帳號 ID 即可：
```json
{
  "accounts": ["帳號A_ID", "帳號B_ID", "帳號C_ID"]
}
```

### Q：影片來源要怎麼標註？
**A：** 在貼文最後加上來源標註：
```
@影片來源：B站 頻道名稱
```
不需要貼連結，這樣就夠了。

### Q：分潤連結要放哪裡？
**A：** 依平台不同：
- **Facebook / IG：** 可放在留言區或貼文內
- **TikTok：** 必須放在影片描述中（不支援留言）

## 📋 發文檢查清單

每次發文前，請確認以下項目：

- [ ] API Key 已設定（`auth:set`）
- [ ] 連線狀態正常（`auth:check`）
- [ ] 帳號清單正確（`accounts:list`）
- [ ] 貼文內容段落間有空行
- [ ] 來源標註格式正確
- [ ] 分潤連結已放置（如需要）
- [ ] 圖片/影片符合平台規格
